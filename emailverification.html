Features of
E-mail Verification

Disposable Status

The email created using a disposable domain is temporary and acts as a proxy. At dper, the disposable domain data is regularly updated from authentic and verified sources.

Common E-mail ID

Check if the email is sent using a commonly available group alias. We have an extensive list of commonly used alias across industries

Name-to-Email matching

The proprietary name-to-email matching algorithm is used to verify if the email belongs to the provided name.

Domain Registration information

It is used to identify domain age, authenticity, and business availability purpose. We use WHOIS and RDAP internet service to fetch the data.

Emailability

Having a valid domain doesn't always ensure it can accept or send email, through mx setup we confirm if the domain can send or receive email.

Name-to-Email matching

Check if a mailbox exists for a given email, it also checks for catch-all cases.

SMTP status code

SMTP status code receive by the SMTP RCPT command, it is used for compliance purposes.

Catch-All

Some Mail Setup configure to accept all emails even the invalid email addresses.It do let us know in case of mailbox response.
